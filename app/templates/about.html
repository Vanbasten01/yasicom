Client Area - Loading Product Content:

Feature: Displaying Products
Explanation: In the client area, the web app loads product content from the database to showcase available items to users. Additionally, the home route (/home) is implemented to retrieve products and deliver a curated list of best deals, sorted from higher to lower prices, utilizing the index1.html template.
Algorithm/Code Snippet: Utilizes Flask routes (/products and /home) to query and retrieve products from the database. The retrieved products are then rendered on the client-facing pages (products.html and home.html), providing users with a dynamic and sorted view of available items.

Admin Dashboard - Managing Store, Products, Orders, and Stock:

Feature: Store Management
Explanation: The admin dashboard provides a comprehensive toolset for managing the online store. It allows tha Admin to add, edit, and remove products, view and process orders, and manage stock levels.
The admin dashboard provides a user-friendly interface for administrators to efficiently manage the store, product catalog, and order processing.

Admin Area - Add a Product:

Feature: Adding a Product
Explanation: Admins can add a new product to the store.
Algorithm/Code Snippet: Implements a form or route for admins to input product details and add a new product to the database.

Admin Area - Product Management:

Feature: Editing, Deleting, and Viewing Products
Explanation: Admin can edit existing products (images, description, cost, product name), delete products, and view detailed information about each product.
Algorithm/Code Snippet: Implements routes and functionality to manage products. This includes:

  • Editing existing products, allowing changes to images, description, cost, and product name.
  • Deleting products from the database.
  • Viewing detailed information about each product.

Admin Area - Stock Management:

Feature: Freezing/Unfreezing Stock
Explanation: Admins can control the availability of a product by freezing or unfreezing its stock.
Algorithm/Code Snippet: Incorporates a mechanism to update the stock status, making the product either available for purchase or temporarily unavailable.

Admin Area - Order Management:

Feature: View Orders, View Order Details, Update Order Status
Explanation: Admins can manage orders with functionalities to view a list of all orders, view detailed information for each order, and update the order status. The default status for an order is 'pending'.
Algorithm/Code Snippet: Implements mechanisms to retrieve and display order information, including details and status updates.
